System Design Documents



Introduction:


  • This class diagram outlines a Fall Detection App that detects when a user falls, confirms the event, and alerts nearby medical workers. It uses GPS to find the user’s location and nearby AEDs, and shares important health details like medical history and DNR status to help responders act quickly and appropriately.


Class Diagram:

  • This class diagram outlines a Fall Detection App that detects when a user falls, confirms the event, and alerts nearby medical workers. It uses GPS to find the user’s location and nearby AEDs, and shares important health details like medical history and DNR status to help responders act quickly and appropriately.


Context Diagram:

  • This context diagram provides a high-level overview of the Fall Detection App and how it interacts with external users and systems. It shows the main components of the app such as the fall detection engine, medical profile manager, AED locator, and alert dispatcher and outlines how users, medical workers, admins, emergency services, and the Google Maps API interact with the system. The diagram highlights the flow of data from user input and sensor data to emergency alerts and location services, illustrating the key roles and responsibilities in ensuring timely and accurate fall response.






Activity Diagram:

  • This activity diagram illustrates the process flow of the Fall Detection App, from detecting a potential fall to initiating an emergency response. It begins with the app collecting sensor and GPS data, running a classification model, and evaluating fall confidence. If a fall is detected, the user is prompted to confirm the alert. Upon confirmation, the app triggers emergency services, sends the user’s location to the server, and displays nearby AEDs. Depending on availability, either nearby medical workers are alerted, or emergency services are contacted directly. The diagram highlights the app’s decision-making and response flow, ensuring timely aid in critical situations.
